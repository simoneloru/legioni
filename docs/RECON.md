# Recon

`legioni init` and `legioni update` run recon: they inspect the current project directory and write a structured profile to `.legioni/project.md`. This file is loaded by opencode at session start via `opencode.json`'s `instructions` field, giving every agent a baseline understanding of what they're working in.

Implementation: `src/core/recon.ts`

## What recon produces

```markdown
# Project Profile

_Generated by legioni recon. Do not edit manually — re-run `legioni update` to refresh._

## Stack

- Language: TypeScript
- Framework: Next.js

## Commands

- Build: `npm run build`
- Test: `npm test`
- Lint/format: `npm run lint`

## Directory Structure

```
bin/
defaults/
docs/
src/:
  adapters/
  commands/
  core/
...
```

## Existing Instructions

_`CLAUDE.md` is present in this repo and is loaded natively by the host — its contents are not duplicated here._
```

The profile has four sections: Stack, Commands, Directory Structure, and Existing Instructions. Commands are omitted if no build/test/lint scripts are detected. The Targeted test line is omitted if no specific test command is available for the ecosystem. The Existing Instructions section is omitted if no AGENTS.md/CLAUDE.md/.cursorrules is present.

## Stack detection support matrix

| Language | Detection method | Framework detection |
|---|---|---|
| TypeScript | `tsconfig.json` present, or `typescript` in deps | Next.js, React, Vue, Svelte, Express, Fastify, Hono |
| JavaScript | `package.json` without TypeScript | same frameworks |
| Python | `pyproject.toml` or `setup.py` | Django, FastAPI, Flask, Apache Airflow (from pyproject.toml content) |
| Go | `go.mod` | none |
| Rust | `Cargo.toml` | none |
| Java (Maven) | `pom.xml` | Maven |
| Java/Kotlin (Gradle) | `build.gradle` or `build.gradle.kts` | Gradle |

If no recognized marker is found, language is reported as `unknown`.

## Command detection

For Node/TypeScript/JavaScript projects, commands come from `package.json` scripts:

| Command type | Detected script | Reported command |
|---|---|---|
| Build | `scripts.build` | `npm run build` |
| Test | `scripts.test` | `npm test` |
| Test (alt) | `scripts["test:run"]` | `npm run test:run` |
| Targeted test | — | `npx vitest run <test-file>` (vitest), `npx jest <test-file>` (jest), or `npm test -- <test-file>` |
| Lint/format | `scripts.lint` | `npm run lint` |
| Lint/format (alt) | `scripts.format` | `npm run format` |

For Go, Rust, Python, and Java, commands are hardcoded based on ecosystem convention.

For Python projects, the test command is always `pytest` and the lint command is always `ruff check .`. The targeted test command is `pytest <test-file>`.

For Maven projects, the targeted test command is `mvn test -Dtest=<TestClass>`. For Gradle, it is `./gradlew test --tests <TestClass>`.

## Existing instructions integration

Recon checks for `AGENTS.md`, `CLAUDE.md`, and `.cursorrules` (in that priority order — first found wins). If one exists, recon writes a one-line note in the profile: the file is present and loaded natively by the host. The file's content is NOT copied into the profile — doing so would cause the rules to be injected twice (once natively by the host, once via `.legioni/project.md`).

## Directory structure

The top-level structure lists non-hidden, non-ignored directories and files. Ignored: `node_modules`, `__pycache__`, `target`, `dist`, `build`, `.git`, `.legioni`.

For one well-known source directory (`src`, `lib`, `app`, `pkg`, `packages` — first found), up to 20 immediate children are also listed.

Hidden directories (`.` prefix) are excluded from the listing. The `.legioni/` directory is also explicitly excluded from the ignore list, so it won't clutter the structure even if visible.

## When to re-run recon

Run `legioni update` when:
- You added or removed a major framework dependency
- You added a new `scripts` entry to `package.json`
- You added `AGENTS.md` or `CLAUDE.md` to the project
- The project directory structure changed significantly

Recon output is not automatically refreshed between sessions. Stale profile data is worse than no profile data — agents may follow outdated commands or look for directories that no longer exist.
